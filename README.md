# Hospital Management System

## 📌 Project Overview
This project focuses on developing a dynamic Hospital Management System that automates hospital operations using a web-based application.

The system provides secure role-based access for **Admin, Doctor, and Patient**. Administrators can manage doctors, patients, appointments, billing, and reports through a centralized dashboard. Doctors can manage appointments, update medical records, and provide prescriptions. Patients can register, book appointments, view prescriptions, and check billing information. The application uses **MongoDB** for real-time data storage and retrieval, ensuring efficient and secure hospital management.

## 🗄️ Database Information

This project uses **MongoDB** as the backend database.

The database stores:

- Patient Details
- Doctor Details
- Appointment Records
- Medical Records
- Prescriptions
- Billing Information
- User Authentication Data

## 🛠️ Project Workflow

### User Authentication
Secure login and registration for Admin, Doctor, and Patient.

### Admin Module
Manage doctors, patients, appointments, billing, and reports.

### Doctor Module
View appointments, manage patient records, and update prescriptions.

### Patient Module
Register, login, book appointments, view prescriptions, and check billing details.

### Database Integration
MongoDB stores and retrieves all hospital-related data in real time.

### Deployment
The application is deployed on **Render** for online access.

## ▶️ How to Run the Project Locally

```bash

```

Open your browser and visit:

